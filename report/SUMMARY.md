# Eliminating Multipliers with K=2 Log/LNS — SkyWater 130 nm (10-bit)

**Function (both designs):** `out = (A·B + C·D) > Vth`, with `A,B,C,D` 10-bit
unsigned (0–1023) and `Vth` a 21-bit programmable threshold (covering the max sum
1023·1023·2 = 2,093,058). Both designs share the same 7-port interface
(`clk, A, B, C, D, Vth, out`) and the same **2-cycle registered-I/O latency**
(62 flip-flops each: 4·10 + 21 input + 1 output).

| | Design 1 — baseline | Design 2 — log / LNS, K=2 |
|---|---|---|
| Method | exact `A*B + C*D`, one comparator | leading-one detector + **2 mantissa bits** → log2; LNS add `max + F(\|Δ\|)`; compare in log domain |
| Multipliers | 2 × `$mul` | **none** |
| Golden spec | `out_exact` (exact integer math) | `out_k1` (the K=2 model — *this* is its "correct" answer) |
| Files | `rtl/mult_detector.v` | `rtl/log_detector.v` + `lod5.v`, `lns_ftable.v`, `lns_add.v` |

The **K=2** helper bits are the two mantissa bits copied below the leading one, so the
log value is `L = 4·⌊log₂v⌋ + (top 2 bits)` in **quarter-log₂ units**. The add-correction
ROM `F(d) = round(4·log2(1+2^−d/4))` (79 entries, d=0…78) is **generated by
`model/model.py` and emitted as `rtl/lns_ftable.v`** — model and hardware cannot drift.
`F = [4,4,3,3,2,2,2,2,1,…]`.

## K=2 accuracy cost (Design 2 vs Design 1)

The 10-bit input space is 1024⁴ ≈ 1.1×10¹² combinations — too large to enumerate — so
the disagreement rate is a **Monte-Carlo estimate over 4,000,000 samples × 11 Vth values**:

**Overall disagreement rate ≈ 2.84 %** (about half the 5.6 % of the earlier K=1 build —
the extra mantissa bit roughly halves the log-quantisation error.)

| Vth | disagree % | | Vth | disagree % |
|----:|-----------:|---|----:|-----------:|
| 0        | 0.000 | | 1,046,529 | 4.050 |
| 1,000    | 0.001 | | 1,048,576 | 6.350 |
| 50,000   | 1.078 | | 1,400,000 | 0.980 |
| 200,000  | 6.900 | | 1,800,000 | 0.030 |
| 500,000  | 11.833| | 2,093,058 | 0.000 |

Peaks in the mid dynamic range and vanishes at the extremes — the accuracy price of
dropping the multiplier. A disagreement is only an RTL bug if RTL ≠ `out_k1` (Phase 4).

## Evidence: no multipliers, no latches, verified

- **No latches** in Design 2 (yosys "No latch inferred" ×14); `CHECK` 0 problems in both.
- **No multipliers in Design 2:** 0 `$mul` cells (Design 1 has 2). F(d) is a small ROM.
- **Verification PASS** (`verif/sim_report.txt`): one testbench streams **58,139** vectors
  through both designs (2-cycle-aligned):
  - Design 1 == `exp_exact` → **0 mismatches, PASS**
  - Design 2 == `exp_k1`    → **0 mismatches, PASS** (bit-exact to the K=2 model)
- **Clean netlists:** both fully mapped to sky130 HD cells, **0 `$`-cells** left.

## Takeaway

**Dropping the two 10-bit multipliers for a K=2 log/LNS detector cuts area by 58 %
(8846 → 3761 µm²) and estimated power by 58 % (0.42× baseline), at a ≈2.8 % disagreement
with exact math.** Vs the 12-bit/K=1 build (−70 %/−74 % at 5.6 %), the area saving is
smaller here because the multiplier is cheaper at 10-bit and K=2 adds a little logic — but
the accuracy roughly doubles.

## Physical design — PnR / DRC / LVS  (NOT run on this host)

Place-and-route, DRC and LVS need OpenROAD/OpenLane + Magic/KLayout + Netgen **and** the
physical sky130 PDK, none of which are installed on the build host — so this repo ships
**logical synthesis + a floorplan estimate**, not a routed layout, and **no DRC/LVS-clean
summary is fabricated**. `pnr/` contains ready-to-run OpenLane configs + a Magic DRC
signoff script to produce the genuine routed GDS + DRC/LVS on a toolchain host; see
`pnr/README.md` and `report/DRC_LVS_STATUS.txt`.
